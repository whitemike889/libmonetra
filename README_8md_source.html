1 About
2 =====
3 
4 LibMonetra is the C library for communicating Main Street Softworks products
5 that utilize the Monetra IP/SSL protocol. It was created by Main Street
6 Softworks for use in their Monetra Payment Engine and associated products. Such
7 as, UniTerm.
8 
9 v8 Changes
10 ----------
11 
12 The v8 version of this library is a ground-up rewrite depending on the mstdlib library. It
13 contains a new API which makes public use of mstdlib objects, therefore to use the new
14 API one must also familiarize themselves with mstdlib. The new API is an event based
15 architecture designed for low-latency and high-volume purposes. The prior API was a
16 polling-based API which was often used in a blocking manner.
17 
18 A full compatibility API for the prior (v3+) interfaces are also available that aim to be
19 fully API and ABI compatible. There may be some slight behavioral differences with the
20 compatibility API that are not expected to cause issues such as automatic reconnects,
21 automatic hardening of returned transaction ids, and always-on thread-safety.
22 
23 As of the v8 release proxy is no longer supported.

25 Building
26 ========
27 
28 CMake is the preferred method to build and should be used whenever possible.
29 However, CMake is not available on all supported platforms. Autotools is also
30 supported for Unix systems. NMake Makefiles are also available for Windows.
31 Autotools and NMake Makefiles are provided only as a fallback when CMake cannot
32 be used. That said, Autotools and NMake Make files can lag behind being updated
33 and there is no timeline on how long they will be supported.
34 
35 The following features are supported by each build system:
36 
37 Feature | CMake | Autotools | NMake Makefiles
38 :--------------------------------------|:-----:|:---------:|:--------------:
39 Shared build | Y | Y | Y
40 Static build | Y | Y | N
41 Disabling building non-base components | Y | Y | N
42 Installation (header and library) | Y | Y | N
43 Disabling header installation | Y | N | N
44 Disabling library installation | Y | N | N
45 Tests | Y | Y | N

56 Dependencies
57 ------------
58 
59 The only dependency is mstdlib available from https://github.com/Monetra/mstdlib/.
60 It should either be compiled and installed prior to libmonetra, or it can be
61 chain built if the source is placed in `thirdparty/mstdlib`.
